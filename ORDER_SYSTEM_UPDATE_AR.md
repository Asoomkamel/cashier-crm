# تحديث نظام الطلبات والمواعيد والتذكيرات

## ملخص التنفيذ

تم تطوير النسخة المرفقة لتضم نظام طلبات موحد داخل قائمة **نظام الطلبات** مع الحفاظ على البنية الحالية للمشروع وبياناته القديمة.

## نظام الطلبات

- تغيير اسم قائمة **الطلبات العاجلة** إلى **نظام الطلبات**.
- إضافة قائمتين فرعيتين في الشريط الجانبي:
  - **الطلبات العاجلة**.
  - **طلبات مكتملة**.
- الضغط على **نظام الطلبات** يفتح تبويب الطلبات العاجلة افتراضيًا.
- يمكن الانتقال بين القائمتين من الشريط الجانبي أو من التبويبات داخل الصفحة.
- الطلبات المكتملة تظهر في قائمة مستقلة، ويمكن إعادتها إلى حالة أخرى من قائمة الحالة عند الحاجة.
- حالات الطلب تظهر للمستخدم بالعربية:
  - قيد الانتظار.
  - تم القبول.
  - قيد التنفيذ.
  - مكتمل.
  - ملغي.
  - مؤجل.
- القيم الداخلية الإنجليزية للحالات لم تُغيّر حتى تبقى قاعدة البيانات وواجهات API والنسخ القديمة متوافقة.

## الطلبات دون موعد زيارة

- الطلب الجديد الذي لا يتم تحديد موعد زيارة له يُحفظ بصورة صريحة دون موعد.
- يظهر الطلب بخلفية حمراء وحدود حمراء.
- يظهر عليه تنبيه **بدون موعد زيارة**.
- تمت إضافة توافق مع البيانات القديمة التي كانت تحفظ وقت إنشاء الطلب داخل حقل التاريخ حتى عند عدم تحديد موعد فعلي.

## التذكيرات التلقائية

- الموعد أو الطلب الذي يحتوي موعد زيارة حقيقيًا يدخل تلقائيًا ضمن التذكيرات عندما يصبح موعده خلال سبعة أيام أو أقل.
- المواعيد المتأخرة تبقى ظاهرة حتى إكمالها أو إلغائها.
- لا ينشأ تذكير زيارة لطلب حالته مكتمل أو ملغي.
- تذكير الزيارة الحالي وتذكير المتابعة المستقبلية لنفس الطلب يُحفظان بمعرفين مختلفين لمنع الاستبدال أو التكرار الخاطئ.
- صفحة التذكيرات تفتح افتراضيًا على فلتر **خلال 7 أيام** حتى تظهر المواعيد القريبة مباشرة.

## تعديل المواعيد

تمت إعادة بناء نافذة إضافة وتعديل الموعد، وتشمل:

- العميل.
- رقم الهاتف.
- موقع العميل.
- تفاصيل الزيارة أو الخدمة.
- الفني.
- تاريخ ووقت الزيارة.
- الحالة.
- الملاحظات.
- إنشاء موعد جديد.
- تعديل موعد موجود بالكامل.
- إعادة الجدولة.
- إكمال الموعد.
- حذف الموعد.
- إظهار الموعد غير المحدد باللون الأحمر.
- تصدير الحالة بالعربية.

## تعديل التذكيرات

يمكن تعديل بيانات التذكير كاملة، بما في ذلك:

- العنوان.
- العميل.
- التاريخ والوقت.
- الأولوية.
- الحالة.
- الدور الموجه إليه.
- الوصف.
- الملاحظات.

التذكيرات التلقائية لا تغيّر الطلب أو الفاتورة الأصلية عند تعديلها؛ بل تحفظ التعديلات كنسخة تخصيص دائمة `override` حتى لا يعيد النظام إنشاء القيمة القديمة فوق تعديل المستخدم.

## تحديث المصطلحات

تم استبدال العبارات الظاهرة المتعلقة بـ **موعد الصيانة** إلى **موعد الزيارة** في:

- نقطة البيع والفاتورة.
- مهام الفني.
- سجل المبيعات.
- الإعدادات.
- التذكيرات.
- رسائل النظام.
- مخططات Excel.

أسماء الحقول الداخلية مثل `nextMaintenanceDate` بقيت كما هي للمحافظة على توافق البيانات القديمة.

## الأمان وملفات البيئة

- لم يتم تضمين `.env.local` في النسخة المعدلة.
- تم تنظيف `.env.example` من القيم السرية واستبدالها بقيم توضيحية.
- يجب إنشاء `.env.local` محليًا أو إعادة استخدام ملف البيئة الخاص بالمستخدم دون رفعه إلى GitHub.
- يوصى بتدوير أي مفاتيح سبق وضعها داخل ملفات مثال أو رفعها إلى مستودع سابق.
